---
name: dm-vs-privileged-actor
description: "DM" describes a user role (use cases, journeys). "Privileged actor" is the technical permission concept (tool access, memory visibility). Multiple actors can be privileged (DM, narrator NPC, deities).
type: feedback
---

- **DM** — a specific user role. Use when describing use cases and user journeys.
- **Privileged character** — meta-level characters that operate outside the
  fiction and define the game rules. Examples: DM, narrator. Privileges belong
  to the CHARACTER, not the actor. Actors inherit the privileges of the
  character they control.
- In-game entities (e.g., deities) are NOT privileged no matter how powerful
  in the story. They are regular characters bound by game rules.
- **Actor** — controls a character, has no privileges of its own. Two general
  types: User (human, via API) and NPC (internal, usually LLM). These are
  just two examples; many specialized subclasses may exist.
